> 最后更新: 2026-06-01
> 版本: v3.0（开场动画 + ESC 暂停菜单 + 结算视频 + 阶段过渡系统 + BGM 系统 + 时长设置）

## 1. 游戏概述

### 1.1 核心概念

**武器角斗**是一款 2D 休闲手游，玩家通过**手绘武器形状 → 选材 → 锻造小游戏 → 试炼场验证**的核心循环来打造独一无二的武器，并在横版竞技场中击败锻造师。

### 1.2 核心卖点

- **画即是武器**：玩家的绘画直接决定武器类型和属性
- **隐藏机制**：多笔画相连形状可触发"复合武器"变形
- **手感反馈**：锤击节奏 + 淬火温控 + 砥砺连击的三阶段锻造体验
- **即时验证**：试炼场立即体验武器效果
- **武器对碰**：武器有碰撞体积和力，格挡产生火花反馈
- **沉浸式演出**：竞技场铁门阶段碎裂 + 开场/结算视频动画 + 心跳节奏音效
- **全服排行榜**：云端积分排行，60秒内击败锻造师挑战最快用时

### 1.3 目标平台

- 移动端（TapTap / iOS / Android）
- PC（TapTap 桌面端）

## 2. 游戏流程

### 2.1 状态机

```
MENU → [开场动画] → DRAW → MATERIAL → FORGE → RESULT → [铁门破碎] → TRIAL → [结算视频] → MENU（循环）
```

| 状态 | 用途 | 关键交互 |
|------|------|----------|
| MENU | 主菜单 | 点击"开始锻造"，设置按键/时长 |
| 开场动画 | 沉浸式开场 | 视频播放，右上角可跳过 |
| DRAW | 绘制武器形状 | 手绘/选模板 → 点击"完成绘制" |
| MATERIAL | 选材（弹幕选择） | 点击飞过的材质方块选中 |
| FORGE | 锻造小游戏（三阶段） | 节奏点击 + 温度控制 + 打磨连击 |
| RESULT | 展示武器属性和品质 | 点击进入试炼 |
| TRIAL | 试炼场横版动作 | 操控角色击败锻造师，60秒限时 |
| 结算视频 | 胜利/失败动画 | 播放对应视频，右上角可跳过 |

### 2.2 状态转换图

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   MENU ─[开场动画]─→ DRAW ─[阶段4]─→ MATERIAL ─[阶段3]─→ FORGE            │
│    ↑                                                          │             │
│    │                                                   [阶段2→1]            │
│    │                                                          ↓             │
│    │                                               RESULT ─[铁门破碎]─→ TRIAL│
│    │                                                                   │    │
│    └──────────────────[结算视频]────────────────────────────────────────┘    │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.3 阶段过渡系统 (PhaseOverlay)

游戏流程通过**竞技场铁门碎裂 + 黑屏字幕**系统串联，营造紧张递进氛围：

| 阶段编号 | 对应状态 | 字幕文案 | 视觉表现 |
|----------|----------|----------|----------|
| 5 | DRAW | "阶段五 — 绘制武器轮廓" | 铁门完整 |
| 4 | MATERIAL | "阶段四 — 选材" | 铁门出现裂纹 + 碎片飞散 |
| 3 | FORGE（锤击） | "阶段三 — 锤击定型" | 裂纹加深 |
| 2 | FORGE（淬火） | "阶段二 — 淬火淬炼" | 裂纹蔓延 |
| 1 | FORGE（砥砺） | "阶段一 — 砥砺磨光" | 即将崩碎 |
| 0 | TRIAL | "试炼 — 以剑证道" | 铁门放大崩碎消散 |

**心跳节奏音效**：随阶段推进，心跳间隔加速（1.4s → 0.45s），营造紧张感。

**碎裂音效**：每次阶段过渡时播放对应等级的碎裂声（crack_1~4 + shatter_final）。

## 3. 开场动画系统

### 3.1 概述

每次开始新一局游戏时，播放开场视频动画，为玩家营造仪式感。

### 3.2 技术实现

| 参数 | 值 | 说明 |
|------|------|------|
| 视频文件 | `video/开场动画.mp4` | 预置在 assets 中 |
| 分辨率 | 1280×720 | 逻辑分辨率 |
| 渲染方式 | 底层 VideoPlayer + NanoVG 纹理 | 非 UI 组件 |
| 淡入时长 | 0.4 秒 | 黑屏→视频 |
| 淡出时长 | 0.5 秒 | 视频→黑屏→游戏 |
| 跳过按钮出现时间 | 播放 2 秒后渐入 | 避免误触 |

### 3.3 跳过机制

- **PC 按键**：ESC 键
- **PC 点击**：右上角 "跳过 >>" 按钮（NanoVG 渲染圆角矩形）
- **移动端触摸**：任意位置触摸即跳过
- **跳过效果**：触发 fadeout → 0.5 秒后进入游戏

### 3.4 交互规则

- 开场动画播放期间**屏蔽所有游戏输入**
- 视频自然播放完毕自动 fadeout
- 视频加载失败时直接跳过（降级处理）

## 4. ESC 暂停菜单系统

### 4.1 概述

在所有游戏环节（主菜单除外）按 ESC 键暂停游戏，弹出暂停菜单。试炼场环节额外展示武器信息卡片。

### 4.2 功能

| 功能 | 说明 |
|------|------|
| 暂停游戏 | 所有游戏逻辑、输入处理停止 |
| 继续游戏 | 关闭菜单，恢复游戏 |
| 返回主界面 | 离开当前状态，回到主菜单 |
| 武器信息卡片 | 仅试炼场：弹窗左侧展示武器名、属性、材质、品质 |

### 4.3 技术实现

- **模块**：`scripts/PauseMenu.lua`
- **拦截位置**：`main.lua` HandleKeyDown 统一拦截 ESC
- **暂停保护**：所有 HandleUpdate、输入 Handler 检查 `PauseMenu.IsVisible()`
- **UI 替换**：暂停时用 `UI.SetRoot()` 覆盖当前 UI，恢复时还原

## 5. 绘制系统 (DRAW)

### 5.1 画布

| 参数 | 值 | 说明 |
|------|------|------|
| 尺寸 | 自适应（基于 UI 面板） | 取面板宽高较小值的 88%~92%，正方形，最小 150px |
| 定位 | 动态居中于中间面板 | 优先 `GetAbsoluteLayout()`，fallback 屏幕计算 |
| 画笔大小 | 4 px | 固定值 |
| 最大采样点数 | 2000 | 单笔画上限 |
| 最小采样间距 | 3 px（平方 = 9） | 防止过密采样 |
| 闭合判定距离 | 20 px（平方 = 400） | 首尾距离小于此值自动闭合 |

### 5.2 笔画数据结构

```lua
stroke = {
    points = { {x=num, y=num}, ... },  -- 画布坐标系
    closed = boolean,                   -- 是否闭合
}
```

### 5.3 预设模板

| 模板 | 描述 | 笔画构成 |
|------|------|----------|
| 直剑 (Sword) | 经典直剑，攻速快 | 剑身+护手一体（单笔画闭合，ar≈2.8） |
| 战斧 (Axe) | 范围大，伤害高 | 斧刃+柄一体（单笔画闭合，斧刃偏右侧不对称） |
| 圆盾 (Shield) | 防御利器 | 外圈（闭合） + 盾心（闭合） |

### 5.4 形状分析器 (Analyzer)

**输入**: 笔画数组
**输出**: `{ type, features, isComposite, shapeCount, aspectRatio, pointCount, closedCount }`

#### 武器类型识别规则（优先级从高到低）

| 类型 | 条件 |
|------|------|
| SHIELD | 闭合数 >= 1，宽高比 0.7~1.4，笔画数 <= 2 |
| AXE | 宽度变化比 > 3.0，宽区域占比 > 0.3，宽高比 0.8~3.5 |
| SPEAR | 宽高比 > 4.0，尖端数 <= 2 |
| SWORD | 宽高比 > 1.8，尖端数 >= 1 |
| HOOK | 尖端数 >= 3 |
| SWORD（默认） | 宽高比 > 1.2 |
| UNKNOWN | 以上均不满足 |

#### 复合武器触发条件（隐藏机制）

同时满足以下条件：
1. 笔画数 >= 2
2. 闭合形状数 >= 1
3. 笔画之间有连接关系（端点间距离 < 30px）

## 7. 锻造系统 (FORGE)

### 7.1 阶段总览

| 阶段 | 时长（可设置） | 玩法 | 满分 | 影响属性 |
|------|------|------|------|----------|
| 锤击 | 5.0 秒 | 节奏点击（5次） | 100 | 品质 |
| 淬火 | 5.0 秒（或松开即止） | 温度控制 | 100 | 品质 |
| 砥砺 | 3.0 秒 | J→K→L 连续按键 | 100 | 攻速加成 |

**最终锻造评分 = (锤击得分 + 淬火得分 + 砥砺得分) / 3**

### 7.2 锤击阶段

- **总时限**: 5 秒（可通过设置面板调整）
- **总锤击次数**: 5 次
- **节奏指示器**: 光标沿横条来回摆动（sin 曲线驱动，周期 1.2 秒）
- **判定区域**:
  - Perfect: 距区域中心 <= 0.10 → 100 分
  - Good: 距区域中心 <= 0.40 → 70 分
  - Miss: 距区域中心 > 0.40 → 20 分
- **交互**: 点击屏幕 / 按空格键（可自定义）

### 7.3 淬火阶段

- **起始温度**: 800~900 度（随机）
- **目标温度**: 300~500 度（随机）
- **冷却**: 按住 -200 度/秒，松开即止
- **评分**: 基于最终温度与目标的差值

### 7.4 砥砺阶段

- **时限**: 3 秒（可通过设置面板调整）
- **操作**: 依次按下 J → K → L（可自定义）完成一次打磨循环
- **得分**: 基于完成循环次数（6+ 次满分）
- **影响**: 攻速加成最高 +30%

### 7.5 音效系统

| 事件 | 音效 | 说明 |
|------|------|------|
| 锤击 | hammer.ogg | 每次锤击播放 |
| 淬火 | quench.ogg | 按住循环播放 |
| 砥砺 | forge_grind.ogg | 按对按键播放 |

## 9. 试炼场 (TRIAL)

### 9.1 核心玩法

**限时横版动作**：60 秒内操控史莱姆角色在平台场景中，使用锻造出的武器击败锻造师（木桩）。击败后提交成绩至全服排行榜。

**操作方式**（默认按键，可自定义）：
| 平台 | 移动 | 跳跃 | 攻击 | 变形 |
|------|------|------|------|------|
| PC | AD / ←→ | 空格 / W / ↑ | J键 / 鼠标左键 | Q键 |
| 移动端 | 左侧触摸区 | 中间触摸区 | 右侧触摸区 | 左下UI按钮 |

### 9.2 玩家参数

| 参数 | 值 | 说明 |
|------|------|------|
| 角色尺寸 | 32×48 px | 碰撞框 |
| 移动速度 | 220 px/s | 攻击时减半 |
| 跳跃初速度 | -420 px/s | 负值=向上 |
| 重力加速度 | 900 px/s² | |
| 主角视觉 | 史莱姆贴图 | 弹性果冻角色，带压扁拉伸 |

### 9.3 场景元素

- **平台**: 8 个单向平台，按比例分布
- **飞行靶子**: 每波 5 个，HP 1000，击破后自动刷新
- **锻造师（木桩）**: HP 9999，永久目标，手持玩家绘制的武器形状

### 9.4 HP 与伤害系统

**设计目标**：普通攻击 6~7 下击杀，重击 3 下击杀

| 武器 | 招式1 伤害 | 招式2 伤害 | 设计定位 |
|------|------------|------------|----------|
| SWORD | 150 | 110 | 均衡型 |
| AXE | 320 | 220 | 爆发型 |
| SPEAR | 130 | 160 | 风筝型 |
| SHIELD | 80 | 120 | 控制型 |
| HOOK | 70 | 240 | 连击型 |
| UNKNOWN | 170 | - | 奇物 |

### 9.5 武器碰撞系统

武器具有碰撞体积（线段模型）和力属性。玩家武器与锻造师武器碰撞时触发格挡：
- 玩家被弹开 + 攻击中断
- 碰撞点火花特效（8方向粒子 + 冲击波环）
- "格挡!" 浮动文字

### 9.6 连击系统

| 参数 | 值 |
|------|------|
| 连击衰减时间 | 2.0 秒 |
| 每次命中得分 | 连击数 × 10 |
| 清场奖励 | 50 分 |

### 9.7 变形系统（复合武器）

- **触发条件**：绘制阶段画出复合武器
- **操作**：Q 键切换形态
- **效果**：拥有两种武器类型的攻击组

### 9.8 试炼结算

| 结束条件 | 结算视频 | 结果 |
|----------|----------|------|
| 锻造师 HP 归零 | 胜利视频 | 显示用时、伤害、连击 → 提交排行榜 |
| 60 秒超时 | 失败视频 | 显示已造成伤害 |
| 玩家被击败 | 失败视频 | 显示已造成伤害 |

玩家可通过设置面板自定义所有 PC 按键，保存在 `keybindings.json`。

| 分类 | 操作 | 默认按键 |
|------|------|----------|

| 设置项 | 默认值 | 说明 |

| 试炼时长 | 60 秒 | 试炼场限时 |
| 选材时长 | 5 秒 | 弹幕选材超时 |
| 锤击时长 | 5 秒 | 锤击阶段时限 |
| 淬火时长 | 5 秒 | 淬火阶段时限 |
| 砥砺时长 | 3 秒 | 砥砺阶段时限 |

### 14.3 核心架构模式

**状态机 + 事件分发**:
- `main.lua` 订阅所有全局事件（仅一次）
- 通过 `STATE_MODULES` 映射表将事件分发到当前活跃状态
- 每个状态模块实现统一接口：`Enter, Leave, Update, Render, OnKeyDown, OnMouseDown, OnMouseUp, OnMouseMove, OnTouchBegin, OnTouchMove, OnTouchEnd, BuildUI`

**输入拦截优先级链**:
```
开场动画（最高优先级，吞掉所有输入）
  → PhaseOverlay 过渡中（屏蔽输入）
    → ESC 暂停菜单拦截
      → PauseMenu 可见时屏蔽
        → 当前状态模块处理
```

**双 NanoVG 上下文**:
- 游戏上下文（renderOrder=0）: 画布/锻造/试炼的自定义绘制
- UI 上下文（renderOrder=999990）: 按钮/文字/布局

### 14.4 跨状态数据流

```lua
gameData_ = {
    strokes = {},          -- [DRAW→] 绘制的笔画数据
    weaponType = "SWORD",  -- [DRAW→] 分析得出的武器类型
    isComposite = false,   -- [DRAW→] 是否为复合武器
    analysis = {},         -- [DRAW→] 完整分析结果
    forgeScore = 0,        -- [FORGE→] 锻造最终评分 (0-100)
    quality = {},          -- [FORGE→] 品质对象 {name, color, threshold}
    material = {},         -- [MATERIAL→] 选中的材质 {id, name, color, atkMod, spdMod, effect, desc}
    weaponData = {},       -- [RESULT→] 最终武器数据（供试炼使用）
}
```
