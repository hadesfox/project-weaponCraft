## 1. 游戏概述

### 1.1 核心概念

**锻造师**是一款 2D 休闲手游，玩家通过**手绘武器形状 → 锻造小游戏 → 试炼场验证**的核心循环来打造独一无二的武器。

### 1.2 核心卖点

- **画即是武器**：玩家的绘画直接决定武器类型和属性
- **隐藏机制**：多笔画相连形状可触发"复合武器"变形
- **手感反馈**：锤击节奏 + 淬火温控的锻造体验
- **即时验证**：试炼场立即体验武器效果

### 1.3 目标平台

- 移动端（TapTap / iOS / Android）
- PC（TapTap 桌面端）

## 2. 游戏流程

### 2.1 状态机

```
MENU → DRAW → FORGE → RESULT → TRIAL → MENU（循环）
```

| 状态 | 用途 | 关键交互 |
|------|------|----------|
| MENU | 主菜单 | 点击"开始锻造" |
| DRAW | 绘制武器形状 | 手绘/选模板 → 点击"完成绘制" |
| FORGE | 锻造小游戏（两阶段） | 节奏点击 + 温度控制 |
| RESULT | 展示武器属性和品质 | 点击任意位置/按钮进入试炼 |
| TRIAL | 试炼场打靶 | 点击挥动武器击破靶子，点击"返回"回主菜单 |

### 2.2 状态转换图


## 3. 绘制系统 (DRAW)

### 3.1 画布

| 参数 | 值 | 说明 |
|------|------|------|
| 尺寸 | 400 x 400 px（逻辑像素） | 居中显示 |
| 画笔大小 | 4 px | 固定值 |
| 最大采样点数 | 2000 | 单笔画上限 |
| 最小采样间距 | 3 px（平方 = 9） | 防止过密采样 |
| 闭合判定距离 | 20 px（平方 = 400） | 首尾距离小于此值自动闭合 |

### 3.2 笔画数据结构

```lua
stroke = {
    points = { {x=num, y=num}, ... },  -- 画布坐标系
    closed = boolean,                   -- 是否闭合
}
```

### 3.3 预设模板

| 模板 | 描述 | 笔画构成 |
|------|------|----------|
| 直剑 (Sword) | 经典直剑，攻速快 | 剑身+护手一体（单笔画闭合，ar≈2.8） |
| 战斧 (Axe) | 范围大，伤害高 | 斧刃+柄一体（单笔画闭合，斧刃偏右侧不对称） |
| 圆盾 (Shield) | 防御利器 | 外圈（闭合） + 盾心（闭合） |

> 注：剑和斧模板设计为单笔画，避免触发复合武器判定。斧通过不对称的宽度分布（斧刃覆盖多个水平带）与剑区分。

### 3.4 形状分析器 (Analyzer)

**输入**: 笔画数组
**输出**: `{ type, features, isComposite, shapeCount, aspectRatio, pointCount, closedCount }`

#### 武器类型识别规则（优先级从高到低）

| 类型 | 条件 |
|------|------|
| SHIELD | 闭合数 >= 1，宽高比 0.7~1.4，笔画数 <= 2 |
| AXE | 宽度变化比 > 3.0，宽区域占比 > 0.3，宽高比 0.8~3.5 |
| SPEAR | 宽高比 > 4.0，尖端数 <= 2 |
| SWORD | 宽高比 > 1.8，尖端数 >= 1 |
| HOOK | 尖端数 >= 3 |
| SWORD（默认） | 宽高比 > 1.2 |
| UNKNOWN | 以上均不满足 |

> AXE 判定说明：使用"宽度带分析"（ComputeWidthProfile），将形状按高度切成 8 个水平带，统计每带的 X 跨度。`widthRatio` = 最宽带/最窄带，`wideSpan` = 宽度超过最大值 40% 的带数占比。斧头的宽刃覆盖较多高度段（wideSpan > 0.3），而剑的护手只占一小段（wideSpan < 0.2）。

#### 尖端检测

- 阈值角度: 60 度
- 最小边长: 3 px（忽略短边的噪声转折）

#### 复合武器触发条件（隐藏机制）

同时满足以下条件：
1. 笔画数 >= 2
2. 闭合形状数 >= 1
3. 笔画之间有连接关系（端点间距离 < 30px）

## 4. 锻造系统 (FORGE)

### 4.1 阶段总览

| 阶段 | 时长 | 玩法 | 满分 |
|------|------|------|------|
| 锤击 | 8.0 秒 | 节奏点击 | 100 |
| 淬火 | 5.0 秒 | 温度控制 | 100 |

**最终锻造评分 = (锤击得分 + 淬火得分) / 2**

### 4.2 锤击阶段

- **节拍生成**: 从 t=1.0s 开始，间隔 = 基础间隔(0.8s) * [0.6, 1.0] 随机
- **判定窗口**:
  - Perfect: ±0.15 秒 → 计 1.0 次命中
  - Good: ±0.3 秒 → 计 0.7 次命中
  - Miss: 超过 Good 窗口 → 计失误
- **得分**: `命中次数 / 总节拍数 * 100`
- **交互**: 点击屏幕 / 按空格键

### 4.3 淬火阶段

- **起始温度**: 800~900 度（随机）
- **目标温度**: 400~500 度（随机）
- **冷却机制**:
  - 自然冷却: -20 度/秒
  - 按住加速: -150 度/秒（额外）
- **评分**:
  - 差值 < 20 度: 100 分
  - 差值 < 50 度: 75 分
  - 差值 < 100 度: 50 分
  - 差值 >= 100 度: 25 分
  - 超时（温度降到 0）: 10 分
- **交互**: 点击/按住 = 加速冷却，松开 = 仅自然冷却

### 4.4 锻造完成过渡

- 淬火完成后延迟 1.5 秒自动进入 RESULT 状态
- 延迟期间忽略玩家输入

## 6. 试炼场 (TRIAL)

### 6.1 核心玩法

**横版动作**：玩家操控角色在带有重力和平台的 2D 横版场景中，左右移动、跳跃，使用锻造出的武器攻击木桩靶子。每次攻击交替使用该武器的两种招式。

**操作方式**：
| 平台 | 移动 | 跳跃 | 攻击 | 变形 |
|------|------|------|------|------|
| PC | AD / ←→ | 空格 / W / ↑ | J键 / 鼠标左键 | Q键 |
| 移动端 | 左侧触摸区 | 中间触摸区 | 右侧触摸区 | 左下UI按钮 |

### 6.2 玩家参数

| 参数 | 值 | 说明 |
|------|------|------|
| 角色尺寸 | 32×48 px | 宽×高 |
| 水平移动速度 | 220 px/s | 攻击时减半 |
| 跳跃初速度 | -420 px/s | 负值=向上 |
| 重力加速度 | 900 px/s² | 下落加速 |
| 最大下落速度 | 600 px/s | 封顶 |
| 地面 Y 位置 | 屏幕高度 × 0.82 | 相对屏幕 |
| 主角贴图 | 像素风锻造师 | 256×256 透明背景 |

### 6.3 场景元素

**平台系统**：
| 参数 | 值 |
|------|------|
| 平台数量 | 3 个 |
| 平台尺寸 | 100×12 px |
| 分布方式 | 水平均分 + 交错高度 |
| 碰撞规则 | 仅从上方下落时站立（单向平台） |

**靶子（木桩）**：
| 参数 | 值 |
|------|------|
| 每波数量 | 5 个 |
| 大小范围 | 25~40 px |
| 分布 | 平台上方 + 地面附近随机 |
| 击破动画 | 碎片飞散 + 击退 + 下落 |
| 重生机制 | 全灭后立即生成新一波 |

### 6.4 攻击系统

#### 招式交替机制

每种武器拥有 1~2 个招式，攻击时循环使用。攻击方向由角色面朝方向决定（左或右）。

#### 攻击类型

| 类型 | 碰撞方式 | 动画表现 |
|------|----------|----------|
| 挥动 (Swing) | 前方矩形 + 扇形弧度 | 武器线段向面朝方向扫过 |
| 突刺 (Thrust) | 点到线段距离 | 水平延伸 → 停留 → 收回 |
| 冲撞 (Charge) | 同挥动 + 玩家前移 | 角色向前冲刺攻击 |

#### 各武器招式（同上一版本配置不变）

| 武器 | 招式1 | 招式2 |
|------|-------|-------|
| **SWORD** 剑 | 横斩 (150°弧, 70px) | 上挑 (100°弧, 60px, 反向) |
| **AXE** 斧 | 劈砍 (90°弧, 80px) | 横扫 (200°弧, 90px) |
| **SPEAR** 矛 | 突刺 (120px超远直线) | 横扫 (160°弧, 100px) |
| **SHIELD** 盾 | 盾击 (强击退) | 冲撞 (前冲40px) |
| **HOOK** 钩 | 钩击 (拉近目标) | 回旋 (360°全向) |
| **UNKNOWN** 奇物 | 挥击 (130°弧, 通用) | - |

### 6.5 碰撞检测（横版）

**挥动类**：检测角色前方范围内的靶子
```
1. 计算靶子到角色攻击原点的距离
2. if 距离 < range + 靶子半径:
   a. 检测靶子是否在面朝方向侧 (dx 方向一致)
   b. 检测垂直距离是否在容差范围内 (|dy| < range * 0.8)
   c. 满足则命中
```

**突刺类**：水平线段检测
```
1. 计算突刺线段: 攻击原点 → 原点 + 面朝方向 × 当前长度
2. 计算靶子圆心到线段最短距离
3. if 距离 < 靶子半径 + 12px → 命中
```

**突刺三段动画**：
- 0%~30%: 快速伸出
- 30%~70%: 停留满距离
- 70%~100%: 快速收回

### 6.6 连击系统

| 参数 | 值 | 说明 |
|------|------|------|
| 连击衰减时间 | 2.0 秒 | 超时后归零 |
| 每次命中得分 | 连击数 × 10 | 连击越高分越多 |
| 清场奖励 | 50 分 | 全灭额外加分 |

### 6.7 武器绘图渲染

**核心机制**：玩家在绘制阶段画出的笔画，直接作为试炼场中武器的外观。

**实现方式**：
1. 绘制阶段的笔画数据（`gameData_.strokes`）传入试炼场
2. 计算所有笔画的整体包围盒 → 归一化到以 (0,0) 为中心的坐标系
3. 根据攻击范围缩放到目标尺寸（约 60px）
4. 攻击时将归一化武器形状绘制在攻击位置，随攻击角度旋转

**攻击时武器渲染**：
| 攻击类型 | 武器位置 | 旋转角度 | 面朝左时 |
|----------|----------|----------|----------|
| 挥动 (Swing) | 攻击弧线 60% 处 | currentAngle + 90° | X 轴翻转镜像（flipX） |
| 突刺 (Thrust) | 突刺线段中点 | 朝右+90° / 朝左-90° | X 轴翻转镜像（flipX） |

> 镜像策略：统一使用 `+π/2` 保证刃端指向外侧，朝左时通过 `nvgScale(-scale, scale)` 翻转 X 轴实现形状水平镜像，确保刃面始终朝外、柄面朝角色。

**无笔画时降级**：若无绘图数据，显示默认剑形线条（竖线 + 护手横线）。

### 6.8 变形系统（复合武器）

**触发条件**：绘制阶段画出 2+ 个相连闭合形状 → 分析器判定为复合武器 → `isComposite = true`

**变形机制**：
- 复合武器拥有两种形态，每种对应不同武器类型的攻击组
- 形态一 = 主识别类型（如 SWORD）
- 形态二 = 互补类型（映射表如下）

**互补类型映射**：
| 主类型 | 互补类型 |
|--------|----------|
| SWORD 剑 | SHIELD 盾 |
| AXE 斧 | HOOK 钩 |
| SPEAR 矛 | SWORD 剑 |
| SHIELD 盾 | SPEAR 矛 |
| HOOK 钩 | AXE 斧 |

**变形操作**：
- PC: Q 键
- 移动端: 左下角 "🔄 变形 (Q)" 按钮
- 攻击中不可变形

**变形反馈**：
- 光环扩散动画（金色圆环 + 6粒子旋转）
- 形态名称闪现文字
- HUD 形态标签实时更新

### 6.9 视觉表现

| 元素 | 表现 |
|------|------|
| 背景 | 深蓝→暗紫渐变 + 远景星星 |
| 地面 | 绿色草地渐变 + 顶部边线 |
| 平台 | 深棕色圆角矩形 + 顶部高亮线 |
| 玩家 | 像素风锻造师贴图 + 方向三角指示 |
| 靶子 | 木桩造型（棕色圆角矩形 + 红色靶心） |
| 武器 (挥动) | 玩家手绘形状随弧度旋转 + 半透明扇形轨迹 + 刃尖发光 |
| 武器 (突刺) | 玩家手绘形状水平延伸 + 轨迹线 + 尖端闪光 |
| 命中效果 | 浮动分数文字上升淡出 |
| 靶子击破 | 5 片碎片放射 + 下落 |
| 连击提示 | 屏幕顶部脉动 "X COMBO!" |
| 变形效果 | 金色光环 + 粒子旋转 + 形态名称 |

### 6.10 HUD（UI 层）

```
│   ├── ForgeState.lua       # 锻造状态
│   ├── ResultState.lua      # 结果展示状态
│   └── TrialState.lua       # 试炼场状态
├── Forge/                   # (预留) 锻造系统扩展
├── Trial/                   # (预留) 试炼系统扩展
└── Weapon/                  # (预留) 武器系统扩展
```

### 8.2 核心架构模式

**状态机 + 事件分发**:
- `main.lua` 订阅所有全局事件（仅一次）
- 通过 `STATE_MODULES` 映射表将事件分发到当前活跃状态
- 每个状态模块实现统一接口：
  ```
  Enter, Leave, Update, Render,
  OnKeyDown, OnMouseDown, OnMouseUp, OnMouseMove,
  OnTouchBegin, OnTouchMove, OnTouchEnd, BuildUI
  ```

**双 NanoVG 上下文**:
- 游戏上下文（`NVG.lua` 管理）: renderOrder=0，用于画布/锻造/试炼的自定义绘制
- UI 上下文（`urhox-libs/UI` 内部）: renderOrder=999990，用于按钮/文字/布局

**渲染层级（从底到顶）**:
1. 游戏 NanoVG（绘制背景、画布、动画）
2. UI 系统（工具栏、按钮、HUD — 需要遮挡 NanoVG 的部分设置 backgroundColor）

### 8.3 跨状态数据流

```lua
gameData_ = {
    strokes = {},          -- [DRAW→] 绘制的笔画数据
    weaponType = "SWORD",  -- [DRAW→] 分析得出的武器类型
    isComposite = false,   -- [DRAW→] 是否为复合武器
    analysis = {},         -- [DRAW→] 完整分析结果
    forgeScore = 0,        -- [FORGE→] 锻造最终评分 (0-100)
    quality = {},          -- [FORGE→] 品质对象 {name, color, threshold}
    weaponData = {},       -- [RESULT→] 最终武器数据（供试炼使用）
}
```

### 8.4 输入处理链路

```
引擎事件 → main.lua(Handler) → GetActiveModule() → State.OnXxx()
```

- 鼠标/触摸坐标统一除以 DPR 转换为逻辑坐标
- DrawState 额外经过 Canvas 的命中测试和坐标转换
