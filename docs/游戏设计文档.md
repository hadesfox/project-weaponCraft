> 最后更新: 2026-05-31

## 1. 游戏概述

### 1.1 核心概念

**锻造师**是一款 2D 休闲手游，玩家通过**手绘武器形状 → 锻造小游戏 → 试炼场验证**的核心循环来打造独一无二的武器。

### 1.2 核心卖点

- **画即是武器**：玩家的绘画直接决定武器类型和属性
- **隐藏机制**：多笔画相连形状可触发"复合武器"变形
- **手感反馈**：锤击节奏 + 淬火温控 + 砥砺连击的三阶段锻造体验
- **即时验证**：试炼场立即体验武器效果
- **武器对碰**：武器有碰撞体积和力，格挡产生火花反馈

### 1.3 目标平台

- 移动端（TapTap / iOS / Android）
- PC（TapTap 桌面端）

## 2. 游戏流程

### 2.1 状态机

```
MENU → DRAW → FORGE → MATERIAL → RESULT → TRIAL → MENU（循环）
```

| 状态 | 用途 | 关键交互 |
|------|------|----------|
| MENU | 主菜单 | 点击"开始锻造" |
| DRAW | 绘制武器形状 | 手绘/选模板 → 点击"完成绘制" |
| FORGE | 锻造小游戏（三阶段） | 节奏点击 + 温度控制 + 打磨连击 |
| MATERIAL | 选材（弹幕选择） | 点击飞过的材质方块选中 |
| RESULT | 展示武器属性和品质 | 点击任意位置/按钮进入试炼 |
| TRIAL | 试炼场打靶 | 点击挥动武器击破靶子，点击"返回"回主菜单 |

### 2.2 状态转换图


## 3. 绘制系统 (DRAW)

### 3.1 画布

| 参数 | 值 | 说明 |
|------|------|------|
| 尺寸 | 自适应（基于 UI 面板） | 取面板宽高较小值的 88%~92%，正方形，最小 150px |
| 定位 | 动态居中于中间面板 | 优先 `GetAbsoluteLayout()`，fallback 屏幕计算 |
| 画笔大小 | 4 px | 固定值 |
| 最大采样点数 | 2000 | 单笔画上限 |
| 最小采样间距 | 3 px（平方 = 9） | 防止过密采样 |
| 闭合判定距离 | 20 px（平方 = 400） | 首尾距离小于此值自动闭合 |

**自适应机制**：画布区域由 UI 面板（`flexGrow=1`，填充顶部工具栏和底部面板之间的剩余空间）决定。Update() 每帧尝试通过 `canvasAreaPanel_:GetAbsoluteLayout()` 获取面板的绝对像素位置和尺寸，计算正方形画布并居中。若 UI 布局未就绪（返回 0 尺寸），自动 fallback 到基于屏幕尺寸的直接计算（预估顶栏 50px + 底栏 110px，取剩余区域 88% 居中）。确保首帧即可渲染画布，适配所有屏幕比例（9:16、4:3、20:9 等）。

### 3.2 笔画数据结构

```lua
stroke = {
    points = { {x=num, y=num}, ... },  -- 画布坐标系
    closed = boolean,                   -- 是否闭合
}
```

### 3.3 预设模板

| 模板 | 描述 | 笔画构成 |
|------|------|----------|
| 直剑 (Sword) | 经典直剑，攻速快 | 剑身+护手一体（单笔画闭合，ar≈2.8） |
| 战斧 (Axe) | 范围大，伤害高 | 斧刃+柄一体（单笔画闭合，斧刃偏右侧不对称） |
| 圆盾 (Shield) | 防御利器 | 外圈（闭合） + 盾心（闭合） |

> 注：剑和斧模板设计为单笔画，避免触发复合武器判定。斧通过不对称的宽度分布（斧刃覆盖多个水平带）与剑区分。

### 3.4 形状分析器 (Analyzer)

**输入**: 笔画数组
**输出**: `{ type, features, isComposite, shapeCount, aspectRatio, pointCount, closedCount }`

#### 武器类型识别规则（优先级从高到低）

| 类型 | 条件 |
|------|------|
| SHIELD | 闭合数 >= 1，宽高比 0.7~1.4，笔画数 <= 2 |
| AXE | 宽度变化比 > 3.0，宽区域占比 > 0.3，宽高比 0.8~3.5 |
| SPEAR | 宽高比 > 4.0，尖端数 <= 2 |
| SWORD | 宽高比 > 1.8，尖端数 >= 1 |
| HOOK | 尖端数 >= 3 |
| SWORD（默认） | 宽高比 > 1.2 |
| UNKNOWN | 以上均不满足 |

> AXE 判定说明：使用"宽度带分析"（ComputeWidthProfile），将形状按高度切成 8 个水平带，统计每带的 X 跨度。`widthRatio` = 最宽带/最窄带，`wideSpan` = 宽度超过最大值 40% 的带数占比。斧头的宽刃覆盖较多高度段（wideSpan > 0.3），而剑的护手只占一小段（wideSpan < 0.2）。

#### 尖端检测

- 阈值角度: 60 度
- 最小边长: 3 px（忽略短边的噪声转折）

#### 复合武器触发条件（隐藏机制）

同时满足以下条件：
1. 笔画数 >= 2
2. 闭合形状数 >= 1
3. 笔画之间有连接关系（端点间距离 < 30px）

## 4. 锻造系统 (FORGE)

### 4.1 阶段总览

| 阶段 | 时长 | 玩法 | 满分 | 影响属性 |
|------|------|------|------|----------|
| 锤击 | 5.0 秒 | 节奏点击（5次） | 100 | 品质 |
| 淬火 | 5.0 秒（或松开即止） | 温度控制 | 100 | 品质 |
| 砥砺 | 3.0 秒 | J→K→L 连续按键 | 100 | 攻速加成 |

**最终锻造评分 = (锤击得分 + 淬火得分 + 砥砺得分) / 3**

### 4.2 锤击阶段

- **总时限**: 5 秒（倒计时，右上角显示，最后3秒变红）
- **总锤击次数**: 5 次（可提前完成）
- **节奏指示器**: 光标沿横条来回摆动（sin 曲线驱动，周期 1.2 秒）
- **判定区域**（随机位置，基于光标与区域中心的距离）:
  - 区域中心每次进入锻造时随机一次（范围 [-0.60, +0.60]），期间不变
  - Perfect: 距区域中心 <= 0.10（金色区域，占比 10%）→ 100 分
  - Good: 距区域中心 <= 0.40（灰色区域，占比 30%）→ 70 分
  - Miss: 距区域中心 > 0.40（暗色区域，占比 60%）→ 20 分
- **冷却**: 每次锤击后 0.3 秒不可再锤（防连点）
- **得分**: `(各次质量分平均) × (完成次数 / 5)`
  - 5 次全部完成：得分 = 各次质量分均值
  - 超时仅完成 3 次：得分 = 均值 × 0.6
- **结束条件**: 5 次锤击完成 **或** 5 秒超时（取先到者）
- **结果展示**: 锤击结束后暂停 1 秒展示得分，然后进入淬火
- **交互**: 点击屏幕 / 按空格键
- **视觉反馈**:
  - 进度圆点颜色：Perfect = 绿色(80,220,100)，Good = 灰色(150,150,170)，Miss = 红色(200,60,60)
  - 光标颜色随位置变化：金色(Perfect区) / 灰色(Good区) / 暗色(Miss区)
  - 横条上黄色矩形标示 Perfect 区，半透明灰色标示 Good 区

### 4.3 淬火阶段

- **起始温度**: 800~900 度（随机）
- **目标温度**: 300~500 度（随机）
- **冷却机制**:
  - 无自然冷却（只有按住时才降温）
  - 按住冷却: -200 度/秒
- **结束条件**（三选一，取先到者）:
  - 玩家**松开按键/触摸**（松开即止）
  - 温度降至 0 度
  - 5 秒超时
- **评分**（最终温度与目标的差值）:
  - 差值 <= 15 度: 100 分（完美）
  - 差值 <= 30 度: 85 分（优秀）
  - 差值 <= 60 度: 65 分（良好）
  - 差值 <= 100 度: 45 分（一般）
  - 差值 <= 150 度: 25 分（较差）
  - 差值 > 150 度: 10 分（很差）
- **交互**: 按住 = 冷却，**松开 = 立即停止淬火（结算评分）**

### 4.4 砥砺阶段

- **时限**: 3 秒（倒计时显示，最后1秒变红）
- **操作**: 依次按下 J → K → L 三个键完成一次打磨循环
- **循环规则**: 完成一轮（J→K→L）后自动从 J 重新开始
- **容错**: 按错键会重置当前轮序列（从 J 重来），不扣分
- **得分**（基于完成打磨次数查表）:
  - 0 次: 10 分
  - 1 次: 30 分
  - 2 次: 50 分
  - 3 次: 65 分
  - 4 次: 80 分
  - 5 次: 90 分
  - 6+ 次: 100 分（满分）
- **影响属性**: 攻速加成 = 砥砺得分 / 100 × 30%（最高 +30% 攻速）
  - 攻速加成在试炼场中降低攻击动画持续时间
  - 例: 砥砺满分 → 攻击动画缩短 30%，出手更快
- **视觉效果**:
  - 背景砂轮持续旋转
  - 按对按键：金色闪光扩散
  - 按错按键：红色闪光提示
  - 按键方框：当前按键金色脉动，已完成按键绿色填充
- **结束条件**: 3 秒超时
- **阶段过渡**: 淬火结束后点击进入砥砺，砥砺结束后延迟 1.5 秒进入 RESULT

### 4.5 音效系统

| 事件 | 音效 | 说明 |
|------|------|------|
| 锤击 | hammer.ogg | 每次锤击播放一次（非循环） |
| 淬火 | quench.ogg | 按住期间循环播放，松开停止 |
| 砥砺 | forge_grind.ogg | 每次按对按键播放一次（磨刀音效） |

**预加载机制（零延迟方案）**：
- 进入锻造状态时预创建 `SoundSource` 组件（复用，不每次新建）
- 以 `gain = 0.0` 静音预播放，触发音频解码管线初始化
- 不调用 `Stop()`（立即 Stop 会取消解码流程，导致首次仍有延迟）
- 实际播放时设置 `gain = 1.0` 后再 `Play()`

### 4.6 锻造完成过渡

- 砥砺完成后延迟 1.5 秒自动进入 RESULT 状态
- 延迟期间忽略玩家输入

## 5. 选材系统 (MATERIAL)

### 5.1 核心玩法

**弹幕选择**：代表各种材质的彩色方块从屏幕右侧飞速飞向左侧，玩家点击/触摸中任意一个方块即表示选中该材质，立即结束选材阶段。

### 5.2 飞行方块参数

| 参数 | 值 | 说明 |
|------|------|------|
| 飞行速度范围 | 700~1200 px/s | 随机，从右→左 |
| 生成间隔 | 0.035 秒 | 高密度弹幕 |
| 方块尺寸 | 85×30 px | 宽×高 |
| 车道数 | 6 | 纵向均匀分布 |
| 车道间距 | 4 px | 上下间隔 |
| 总时限 | 5.0 秒 | 超时自动随机选择 |

### 5.3 交互规则

- **点击方块** → 立即选中该方块对应的材质 → 1.2 秒结果展示 → 进入 RESULT
- **超时** → 自动随机选择一个材质（降级策略）
- 方块渲染使用该材质的专属颜色填充 + 亮色描边 + 白色材质名称文字

### 5.4 材质一览

| ID | 名称 | 颜色 RGB | 攻击修正 | 攻速修正 | 效果 | 描述 |
|------|------|----------|----------|----------|------|------|
| black_iron | 黑铁 | (60,60,80) | +20% | -10% | heavy_blow | 重击：击退距离+50% |
| mithril | 秘银 | (180,220,255) | -10% | +25% | agile | 灵动：攻击时移速不减 |
| obsidian | 黑曜石 | (30,10,50) | +40% | 0% | shatter | 碎裂：自身受击+20% |
| blood_ore | 血矿 | (180,30,30) | -15% | 0% | lifesteal | 嗜血：伤害15%回血 |
| radiant_meteor | 陨星 | (100,255,80) | -30% | 0% | burn | 灼烧：每秒2%真伤 |
| inverse_alloy | 逆鳞 | (200,160,50) | -20% | 0% | thorns | 反震：格挡反伤100 |
| spirit_wood | 灵木 | (80,200,120) | 0% | 0% | growth | 成长：连击+5%伤害(上限50%) |

### 5.5 材质效果详解（试炼场生效）

#### atkMod / spdMod（通用属性修正）

所有材质均有 `atkMod`（攻击力修正）和 `spdMod`（攻速修正）两项基础属性：
- **atkMod**: 乘算修正伤害 → `最终伤害 = 基础伤害 × (1 + atkMod)`
- **spdMod**: 乘算修正攻击持续时间 → `最终持续时间 = 基础持续时间 × (1 - speedBonus - spdMod)`
  - 正值加速（缩短攻击间隔），负值减速

#### 材质特效（effect）

| 效果 ID | 名称 | 实装位置 | 机制详解 |
|---------|------|----------|----------|
| heavy_blow | 重击 | HitTarget | 命中时击退距离 × 1.5 |
| agile | 灵动 | UpdatePlayerPhysics | 攻击期间不清零移动速度，可边打边走 |
| shatter | 碎裂 | CheckDummyAttackHitPlayer | 被木桩命中时击退 × 1.2（代价型材质） |
| lifesteal | 嗜血 | HitTarget + CheckDummyAttackHitPlayer | 命中时伤害 15% 转为护盾值；被击时护盾抵消 30% 击退 |
| burn | 灼烧 | Update 主循环 | 每秒对木桩造成 DummyHP × 2% 真实伤害（独立于攻击） |
| thorns | 反震 | CheckWeaponClash | 格挡武器碰撞时对木桩造成 100 固定伤害 |
| growth | 成长 | HitTarget | 每次命中 +5% 伤害加成（乘算），上限 50% |

### 5.6 跨状态数据传递

选中材质后写入 `gameData_.material`：
```lua
gameData_.material = {
    id = "black_iron",
    name = "黑铁",
    color = { 60, 60, 80 },
    atkMod = 0.20,
    spdMod = -0.10,
    effect = "heavy_blow",
    desc = "重击：击退距离+50%",
}
```

试炼场 Enter 时读取并缓存为局部变量 `materialEffect_`、`materialAtkMod_`、`materialSpdMod_`。


## 7. 试炼场 (TRIAL)

### 7.1 核心玩法

**横版动作**：玩家操控角色在带有重力和平台的 2D 横版场景中，左右移动、跳跃，使用锻造出的武器攻击木桩靶子。每次攻击交替使用该武器的两种招式。

**操作方式**（默认按键，可自定义）：
| 平台 | 移动 | 跳跃 | 攻击 | 变形 |
|------|------|------|------|------|
| PC | AD / ←→ | 空格 / W / ↑ | J键 / 鼠标左键 | Q键 |
| 移动端 | 左侧触摸区 | 中间触摸区 | 右侧触摸区 | 左下UI按钮 |

> 所有 PC 按键可通过主菜单设置界面自定义（详见第 13 章 按键绑定系统）。

### 7.2 玩家参数

| 参数 | 值 | 说明 |
|------|------|------|
| 角色尺寸 | 32×48 px | 宽×高（碰撞框） |
| 水平移动速度 | 220 px/s | 攻击时减半 |
| 跳跃初速度 | -420 px/s | 负值=向上 |
| 重力加速度 | 900 px/s² | 下落加速 |
| 最大下落速度 | 600 px/s | 封顶 |
| 地面 Y 位置 | 屏幕高度 × 0.82 | 相对屏幕 |
| 主角视觉 | 史莱姆贴图 | 弹性果冻角色，带压扁拉伸 |

#### 史莱姆角色渲染 (Trial/Slime.lua)

**视觉表现**：使用贴图渲染（`image/史莱姆_*.png`），以椭圆裁剪显示，底部贴地。

**弹性动画（压扁/拉伸）**：
- **落地压扁**：落地瞬间 stretchX 增大、stretchY 减小，冲击强度由上一帧竖直速度决定
- **起跳拉伸**：起跳瞬间 stretchX 减小、stretchY 增大
- **恢复弹簧**：通过 springK=12.0 弹簧插值回归原始尺寸
- **体积守恒**：`stretchX * stretchY ≈ 1.0`，偏差超过 0.01 自动修正
- **移动节律**：地面移动时产生频率为 12Hz 的微弹跳效果
- **待机微弹**：站立不动时以 3Hz 频率轻微起伏（幅度 4%）

**关键常量**：
| 参数 | 值 | 说明 |
|------|------|------|
| SLIME_BASE_SIZE | 40 px | 基础直径（按 physScale 缩放） |
| SQUASH_DURATION | 0.20 s | 落地压扁持续时间 |
| STRETCH_AMOUNT | 0.35 | 拉伸/压缩最大幅度 |
| BOUNCE_ANIM_SPEED | 12 | 移动弹跳频率 |

### 7.6 碰撞检测（横版）

**挥动类**：检测角色前方范围内的靶子
```
1. 计算靶子到角色攻击原点的距离
2. if 距离 < range + 靶子半径:
   a. 检测靶子是否在面朝方向侧 (dx 方向一致)
   b. 检测垂直距离是否在容差范围内 (|dy| < range * 0.8)
   c. 满足则命中
```

**突刺类**：水平线段检测
```
1. 计算突刺线段: 攻击原点 → 原点 + 面朝方向 × 当前长度
2. 计算靶子圆心到线段最短距离
3. if 距离 < 靶子半径 + 12px → 命中
```

**突刺三段动画**：
- 0%~30%: 快速伸出
- 30%~70%: 停留满距离
- 70%~100%: 快速收回

### 7.8 武器碰撞系统

#### 概述

武器具有**碰撞体积**（线段模型）和**力属性**（大小 + 方向）。当玩家武器与木桩武器碰撞时，触发格挡反馈：玩家被弹开、攻击中断、火花特效。

#### 碰撞体模型

每把武器建模为一条**线段**（从根部到尖端）+ 碰撞宽度：

```
武器碰撞体 = {
    rootX, rootY,      -- 根部世界坐标
    tipX, tipY,        -- 尖端世界坐标
    width,             -- 碰撞宽度（胶囊体半径）
    force,             -- 力的大小（击退力度）
    forceDir,          -- 力的方向（1=向右，-1=向左）
}
```

#### 木桩武器参数

| 参数 | 值 | 说明 |
|------|------|------|
| angle | -0.3 rad | 略微上扬向右（约 -17°） |
| length | 50 px | 基础长度（乘以 physScale） |
| width | 8 px | 碰撞宽度（乘以 physScale） |
| force | 12 | 击退力度 |
| forceDir | 1 | 力方向：向右 |
| localOffsetY | -0.6 | 根部位于木桩高度 60% 处 |
| **可见性** | **仅攻击时** | 木桩未攻击时武器完全隐藏 |
| **格挡能力** | **仅攻击时** | 武器隐藏时无碰撞体，玩家攻击不会被格挡 |

**木桩武器渲染**：使用玩家绘制的武器形状（`RenderWeaponShape`），颜色偏暗红以区分于玩家武器。仅在木桩攻击动画期间显示，附带挥砍轨迹特效（半透明尾迹线）。平时武器完全隐藏，玩家可自由攻击而不会被格挡。

#### 玩家武器碰撞体

玩家攻击时动态生成碰撞体：
- **挥动攻击**：碰撞线段从攻击原点到当前弧度位置的刃尖
- **突刺攻击**：碰撞线段从攻击原点沿面朝方向延伸到当前突刺长度
- **碰撞宽度**: 12 px（乘以 physScale）
- **力**: 继承当前招式的 `knockback` 值

#### 碰撞检测算法

使用**线段-线段最短距离**检测：
1. 对两条线段各取 3 个采样点（两端点 + 中点）
2. 计算每个采样点到对方线段的最短距离（共 6 次点-线段距离计算）
3. 取 6 个距离的最小值作为两线段最短距离
4. 若距离 < (玩家武器宽度 + 木桩武器宽度) / 2 → 判定碰撞

#### 碰撞反馈

| 效果 | 详情 |
|------|------|
| 玩家击退 | `vx = 推向 × force × physScale × 8`，`vy = -force × physScale × 3` |
| 攻击中断 | `attacking_ = false`，当前攻击立即终止 |
| 木桩抖动 | `hitAnim = 0.6`，方向为玩家面朝方向 |
| 碰撞特效 | 碰撞点处火花扩散（8方向粒子 + 金色闪光 + 冲击波环） |
| 文字反馈 | 碰撞点上方浮动 "格挡!" 文字（金色，1秒淡出） |
| 冷却时间 | 0.4 秒内不再检测（防止连续触发） |

#### 特效动画

碰撞火花由 `weaponClashAnim_`（1.0→0，衰减速度 ×3）驱动：
- **中心闪光**：金色圆形，半径 8+扩展
- **火花粒子**：8 个方向飞散，颜色 RGB(255,180,50)，大小随动画衰减
- **冲击波环**：白色描边圆环，半径随时间扩展，透明度随时间衰减

### 7.9 武器绘图渲染

**核心机制**：玩家在绘制阶段画出的笔画，直接作为试炼场中武器的外观。

**实现方式**：
1. 绘制阶段的笔画数据（`gameData_.strokes`）传入试炼场
2. 计算所有笔画的整体包围盒 → 归一化到以 (0,0) 为中心的坐标系
3. 根据攻击范围缩放到目标尺寸（约 60px）
4. 攻击时将归一化武器形状绘制在攻击位置，随攻击角度旋转

**攻击时武器渲染**：
| 攻击类型 | 武器位置 | 旋转角度 | 面朝左时 |
|----------|----------|----------|----------|
| 挥动 (Swing) | 攻击弧线 60% 处 | currentAngle + 90° | X 轴翻转镜像（flipX） |
| 突刺 (Thrust) | 突刺线段中点 | 朝右+90° / 朝左-90° | X 轴翻转镜像（flipX） |

> 镜像策略：统一使用 `+π/2` 保证刃端指向外侧，朝左时通过 `nvgScale(-scale, scale)` 翻转 X 轴实现形状水平镜像，确保刃面始终朝外、柄面朝角色。

**无笔画时降级**：若无绘图数据，显示默认剑形线条（竖线 + 护手横线）。

### 9.3 核心架构模式

**状态机 + 事件分发**:
- `main.lua` 订阅所有全局事件（仅一次）
- 通过 `STATE_MODULES` 映射表将事件分发到当前活跃状态
- 每个状态模块实现统一接口：
  ```
  Enter, Leave, Update, Render,
  OnKeyDown, OnMouseDown, OnMouseUp, OnMouseMove,
  OnTouchBegin, OnTouchMove, OnTouchEnd, BuildUI
  ```

**双 NanoVG 上下文**:
- 游戏上下文（`NVG.lua` 管理）: renderOrder=0，用于画布/锻造/试炼的自定义绘制
- UI 上下文（`urhox-libs/UI` 内部）: renderOrder=999990，用于按钮/文字/布局

**渲染层级（从底到顶）**:
1. 游戏 NanoVG（绘制背景、画布、动画）
2. UI 系统（工具栏、按钮、HUD — 需要遮挡 NanoVG 的部分设置 backgroundColor）

### 9.4 跨状态数据流

```lua
gameData_ = {
    strokes = {},          -- [DRAW→] 绘制的笔画数据
    weaponType = "SWORD",  -- [DRAW→] 分析得出的武器类型
    isComposite = false,   -- [DRAW→] 是否为复合武器
    analysis = {},         -- [DRAW→] 完整分析结果
    forgeScore = 0,        -- [FORGE→] 锻造最终评分 (0-100)
    quality = {},          -- [FORGE→] 品质对象 {name, color, threshold}
    material = {},         -- [MATERIAL→] 选中的材质 {id, name, color, atkMod, spdMod, effect, desc}
    weaponData = {},       -- [RESULT→] 最终武器数据（供试炼使用）
}
```

### 9.5 输入处理链路

```
引擎事件 → main.lua(Handler) → GetActiveModule() → State.OnXxx()
```

- 鼠标/触摸坐标统一除以 DPR 转换为逻辑坐标
- DrawState 额外经过 Canvas 的命中测试和坐标转换

## 13. 按键绑定系统

### 13.1 概述

玩家可通过主菜单的**设置界面**（全屏模式）自定义所有 PC 按键绑定。设置保存在本地文件 `keybindings.json`，下次启动自动加载。

### 13.2 可配置操作

| 分类 | 操作 ID | 默认按键 | 说明 |
|------|---------|----------|------|
| 试炼场 | move_left | A / ← | 左移 |
| 试炼场 | move_right | D / → | 右移 |
| 试炼场 | move_down | S / ↓ | 下蹲/下跳 |
| 试炼场 | jump | 空格 / W / ↑ | 跳跃 |
| 试炼场 | attack1 | J | 攻击 1 |
| 试炼场 | attack2 | K | 攻击 2 |
| 试炼场 | transform | Q | 变形 |
| 锻造 | forge_hit | 空格 | 锤击/淬火 |
| 锻造 | grind1 | J | 砥砺 1 |
| 锻造 | grind2 | K | 砥砺 2 |
| 锻造 | grind3 | L | 砥砺 3 |
| 绘图 | draw_undo | Z | 撤销 |
| 绘图 | draw_clear | X | 清除 |

### 13.3 技术实现

- **模块**: `scripts/KeyBindings.lua`
- **存储**: JSON 文件 `keybindings.json`（引擎文件沙箱内）
- **API**:
  - `KeyBindings.IsKey(actionId, key)` — 检查按键是否对应某操作
  - `KeyBindings.IsDown(actionId)` — 检查操作是否正在按下
  - `KeyBindings.GetDisplayText(actionId)` — 获取绑定显示文本
  - `KeyBindings.ResetToDefault()` — 恢复默认
- **容错**: 文件不存在时静默使用默认绑定，不报错

### 13.4 设置界面

- **入口**: 主菜单右上角齿轮图标
- **呈现**: 全屏覆盖面板（深色半透明背景）
- **内容**: 分类展示所有可配置操作 + 当前绑定按键 + 重置按钮
- **交互**: 点击按键槽 → 等待按键输入 → 绑定新键
